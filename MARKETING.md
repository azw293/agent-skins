1. Browse skins at skins.sh
2. Install with `npx skins add agent-skins --skin X`
3. Works with Claude Code, Cursor, Codex CLI, Gemini CLI

Each skin is a SKIN.md file — YAML frontmatter with metadata + markdown body with the persona prompt. Open standard, model-agnostic, MIT licensed.

## 🎯 Positioning & messaging framework

### Core positioning
**skins.sh is the open marketplace for AI agent character skins.**

### One-liner
Give your AI agent a personality.

### Elevator pitch (30 seconds)
"Skills tell your AI agent what it can do. Skins tell it who it is. We built the open marketplace for AI agent character skins — think jack-sparrow, yoda, sherlock-holmes as portable persona prompts you install in one command. It works with Claude, Cursor, Codex, GPT-4. Free, open source, MIT."

### Target audiences

**Primary:**
- Developers using Claude Code, Cursor, or Codex CLI who want more engaging agents
- AI builders who personalize their chatbots and assistants
- The skills.sh / agent-skills community

**Secondary:**
- Productivity / personal development crowd (stoic-philosopher, samurai-warrior)
- Developers who find AI interactions boring and want something more interesting
- People building AI products who want character variety

### Differentiation from competitors

| Competitor | How we're different |
|------------|---------------------|
| Character.ai | They're a chat product. We're infrastructure for agents. |
| ChatGPT custom instructions | Not portable, not open, not community-shared |
| Skills marketplaces | We do personalities, not capabilities — complementary |
| System prompt templates | We have a standard format, CLI tooling, and a community |

### Viral hooks
1. "Your AI agent is boring. Here's how to fix it." (problem)
2. "I made Sherlock Holmes debug my code" (story)
3. "Jack Sparrow just refactored my codebase" (funny specific)
4. Screenshots of actual skin outputs (social proof)
5. The concept of skins from gaming applied to AI (familiar framing)
